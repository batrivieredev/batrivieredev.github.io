Compilation et Liaisons
 

Compilation et Liens en C

1. Introduction

La compilation et le processus de liaison sont essentiels pour transformer le code source écrit en C en un programme exécutable. Ce processus se compose de plusieurs étapes.

2. Étapes de la Compilation

  1. Prétraitement : Le préprocesseur traite les directives de prétraitement (commençant par #). Cela inclut l'inclusion de fichiers d'en-tête et la définition de macros.
  2. Compilation : Le compilateur traduit le code source en code objet, qui est un code machine non exécutable.
  3. Édition de liens : Le linker combine les fichiers objets et les bibliothèques pour créer un exécutable.

3. Exemple de Code

Considérons un simple programme C qui utilise une fonction définie dans un autre fichier :

Fichier : main.c

#include <stdio.h>
#include "utils.h"

int main() {
    int result = add(5, 3);
    printf("La somme est : %d\n", result);
    return 0;
}

Fichier : utils.c

#include "utils.h"

int add(int a, int b) {
    return a + b;
}

Fichier : utils.h

int add(int a, int b);

4. Compilation et Liaison

Pour compiler et lier ces fichiers, vous pouvez utiliser les commandes suivantes dans un terminal :

gcc -c utils.c   # Compile utils.c en utils.o
gcc -c main.c     # Compile main.c en main.o
gcc -o mon_programme main.o utils.o  # Lie main.o et utils.o pour créer l'exécutable

5. Exécution du Programme

Après avoir compilé et lié le programme, vous pouvez l'exécuter avec la commande :

./mon_programme

Conclusion

La compilation et l'édition de liens sont des étapes cruciales dans le développement de programmes en C. Comprendre ces processus vous aidera à mieux gérer votre code et à résoudre les problèmes liés à la compilation.
