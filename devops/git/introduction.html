Introduction à Git et GitHub

Git est un système de contrôle de version distribué, tandis que GitHub est une plateforme d'hébergement de code qui utilise Git.

Commandes Git essentielles

# Configuration initiale
git config --global user.name "John Doe"
git config --global user.email "john@example.com"

# Initialisation et clonage
git init                   # Nouveau dépôt
git clone repo_url         # Cloner un dépôt

# Gestion des modifications
git status                # État des fichiers
git add fichier          # Ajouter des fichiers
git add .                # Ajouter tous les fichiers
git commit -m "message"  # Valider les changements

# Branches
git branch               # Lister les branches
git branch nom          # Créer une branche
git checkout nom        # Changer de branche
git checkout -b nom     # Créer et changer
git merge branche       # Fusionner une branche

# Remote
git remote add origin url  # Ajouter un dépôt distant
git push origin main      # Envoyer les modifications
git pull origin main      # Récupérer les modifications
git fetch origin         # Mettre à jour les références

Workflow GitHub

# Fork et Clone
# 1. Fork sur GitHub
# 2. Cloner localement
git clone https://github.com/user/repo.git

# Créer une branche pour la fonctionnalité
git checkout -b feature/nouvelle-fonc

# Développer et commiter
git add .
git commit -m "Ajout nouvelle fonctionnalité"

# Mettre à jour depuis upstream
git remote add upstream original_repo_url
git fetch upstream
git rebase upstream/main

# Pousser les modifications
git push origin feature/nouvelle-fonc

# Créer une Pull Request sur GitHub
# 1. Aller sur GitHub
# 2. Cliquer "New Pull Request"
# 3. Sélectionner les branches
# 4. Décrire les modifications

Fonctionnalités avancées

Git avancé

# Rebase interactif
git rebase -i HEAD~3

# Cherry-pick
git cherry-pick commit_hash

# Reset et Revert
git reset --hard HEAD~1  # Annuler dernier commit
git revert commit_hash   # Créer commit inverse

# Stash
git stash                # Sauvegarder modifications
git stash list          # Lister les stash
git stash pop           # Appliquer et supprimer
git stash apply         # Appliquer sans supprimer

# Tags
git tag v1.0.0           # Créer un tag
git push origin v1.0.0   # Pousser un tag

GitHub Actions

# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18.x'
    - name: Install dependencies
      run: npm ci
    - name: Run tests
      run: npm test
    - name: Build
      run: npm run build
