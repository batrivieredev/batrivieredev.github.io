Kubernetes - Bonnes Pratiques

Guide des meilleures pratiques pour concevoir, déployer et maintenir des applications sur Kubernetes.

Sécurité

✅ Bonnes pratiques

# RBAC strict
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: app-role
rules:
- apiGroups: [""]
  resources: ["pods", "services"]
  verbs: ["get", "list", "watch"]

# Container sécurisé
securityContext:
  runAsNonRoot: true
  runAsUser: 1000
  readOnlyRootFilesystem: true
  capabilities:
    drop: ["ALL"]

# Network Policy
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress

❌ À éviter

# Éviter les privilèges root
securityContext:
  privileged: true  # ❌

# Ne pas exposer les ports inutilement
ports:
- containerPort: 22  # ❌ SSH

# Pas de secrets en clair
env:
- name: DB_PASSWORD
  value: "password123"  # ❌

Resources et limites

# Définition des ressources
resources:
  requests:
    memory: "64Mi"
    cpu: "250m"
  limits:
    memory: "128Mi"
    cpu: "500m"

# LimitRange par namespace
apiVersion: v1
kind: LimitRange
metadata:
  name: resource-limits
spec:
  limits:
  - type: Container
    default:
      memory: "512Mi"
      cpu: "500m"
    defaultRequest:
      memory: "256Mi"
      cpu: "250m"
    max:
      memory: "1Gi"
      cpu: "1"
    min:
      memory: "64Mi"
      cpu: "100m"

# ResourceQuota
apiVersion: v1
kind: ResourceQuota
metadata:
  name: compute-quota
spec:
  hard:
    requests.cpu: "4"
    requests.memory: 4Gi
    limits.cpu: "8"
    limits.memory: 8Gi

Haute disponibilité

# Anti-affinité des pods
spec:
  affinity:
    podAntiAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
      - labelSelector:
          matchExpressions:
          - key: app
            operator: In
            values:
            - web-server
        topologyKey: "kubernetes.io/hostname"

# Readiness et Liveness Probes
spec:
  containers:
  - name: app
    readinessProbe:
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 10
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 15
      periodSeconds: 20

# PodDisruptionBudget
apiVersion: policy/v1
kind: PodDisruptionBudget
metadata:
  name: app-pdb
spec:
  minAvailable: 2
  selector:
    matchLabels:
      app: web-server

Points clés :

  • Répartir les pods sur plusieurs nœuds
  • Configurer des probes adaptées
  • Définir des budgets de perturbation
  • Mettre en place des backups

Monitoring et logging

# Labels pour le monitoring
metadata:
  labels:
    app.kubernetes.io/name: myapp
    app.kubernetes.io/instance: prod
    app.kubernetes.io/version: "1.0.0"
    app.kubernetes.io/component: frontend
    app.kubernetes.io/part-of: web-app
    app.kubernetes.io/managed-by: helm

# Annotations pour Prometheus
metadata:
  annotations:
    prometheus.io/scrape: "true"
    prometheus.io/port: "8080"
    prometheus.io/path: "/metrics"

# Logging sidecar
spec:
  containers:
  - name: app
    # ...
  - name: logging
    image: fluent/fluent-bit
    volumeMounts:
    - name: logs
      mountPath: /var/log

GitOps et CI/CD

# ArgoCD Application
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: myapp
spec:
  project: default
  source:
    repoURL: https://github.com/org/app.git
    targetRevision: HEAD
    path: k8s
  destination:
    server: https://kubernetes.default.svc
    namespace: myapp

# Kustomization
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
namespace: myapp
resources:
- deployment.yaml
- service.yaml
- ingress.yaml
commonLabels:
  env: production
configMapGenerator:
- name: app-config
  literals:
  - API_URL=https://api.example.com

Conventions de nommage

  • Namespaces : {env}-{team}-{app} (ex: prod-team1-webapp)
  • Pods/Deployments : {app}-{component}-{id} (ex: webapp-frontend-v1)
  • Services : {app}-{component}-svc (ex: webapp-frontend-svc)
  • ConfigMaps/Secrets : {app}-{type} (ex: webapp-config, webapp-secrets)
  • PVCs : {app}-{usage}-pvc (ex: webapp-data-pvc)
