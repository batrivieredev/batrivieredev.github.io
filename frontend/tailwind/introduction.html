Introduction à Tailwind CSS

Tailwind CSS est un framework CSS utilitaire qui permet de construire rapidement des interfaces personnalisées sans quitter votre HTML.

Installation

// Avec npm
npm install -D tailwindcss
npx tailwindcss init

// tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{html,js,vue,jsx,tsx}"
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
// styles.css
@tailwind base;
@tailwind components;
@tailwind utilities;

Concepts fondamentaux

1. Responsive Design

<div class="w-full md:w-1/2 lg:w-1/3">
  Responsive width
</div>

<div class="hidden md:block">
  Visible on medium screens and up
</div>

2. États

<button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded focus:outline-none focus:shadow-outline">
  Button
</button>

<input class="border-2 focus:border-blue-500 focus:ring"
  type="text">

3. Dark Mode

<div class="bg-white dark:bg-gray-800 text-gray-900 dark:text-white">
  Auto dark mode
</div>

Flexbox & Grid

// Flexbox
<div class="flex justify-between items-center">
  <div>Item 1</div>
  <div>Item 2</div>
</div>

// Grid
<div class="grid grid-cols-3 gap-4">
  <div>1</div>
  <div>2</div>
  <div>3</div>
</div>

Personnalisation

// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        'brand': {
          light: '#FAFAFA',
          DEFAULT: '#1a73e8',
          dark: '#1557B0',
        }
      },
      spacing: {
        '128': '32rem',
      },
      fontFamily: {
        sans: ['Roboto', 'sans-serif'],
      }
    }
  }
}

Bonnes pratiques

✅ À faire

  • Utiliser des composants pour le code répétitif
  • Exploiter @apply pour les styles communs
  • Configurer purge CSS en production
  • Suivre une convention de nommage

❌ À éviter

  • Trop de classes personnalisées
  • Styles en ligne avec !important
  • Duplication de code
  • Surcharger les classes utilitaires
